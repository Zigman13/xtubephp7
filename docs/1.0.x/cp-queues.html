Conversion and Thumbnail Generation Queues
Table of Contents
Overview
TubeX uses a queue system to manage converting videos and generating thumbnails for videos. This queue system ensures that your server does not get overloaded by many video conversion or thumbnailing processes running at the same time. Instead each video that needs to be converted or thumbnailed will be added to a queue and processed in the order it was added.
Queue Interfaces
Through the TubeX control panel you can view and manage the queues. When you access either the Conversion Queue or Thumbnail Queue functions in the Videos menu, a dialog window will appear that looks like this:


The Queue Details will include the following:
    Queued Videos - The total number of items in the queue waiting to be processed
    Queue Processor - The status of the queue processor, either Running or Stopped
    Videos Processed - The total number of videos that have been successfully processed
    Average Time - The average amount of time it takes to process a video in the queue
    Estimated Time - The estimated amount of time remaining required to process all items in the queue
    Last Started - The date and time the queue process was last started
    Failed - The number of videos that were not able to be processed successfully
    Average Wait - The average amount of time an item waits in the queue before being processed
The Control Buttons available at the bottom of the queue interfaces are:
    Start - Press this button to start processing the items in the queue. This button will only be available when the queue is not running.

    It may not be possible to start the queue process through the control panel interfaces on all servers. Your server will need to have PHP safe_mode disabled, the PHP shell_exec() function available, and the CLI or CGI version of PHP available on the server.

    Stop - Press this button to stop processing the items in the queue. This button will only be available when the queue is already running.
    Clear Queue - Press this button to remove all items from the queue. This button will only be available when the queue is not running.
    Close - Press this button to close the queue interface. This will not have any affect on the operation of the queue (i.e. it will continue running even if you close this interface)
