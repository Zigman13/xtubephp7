Managing Custom Category Fields
Managing custom category fields is done by accessing the Categories > Custom Fields > Search Custom Category Fields function in the control panel menu. From this interface you can add new custom category fields and search, edit, or delete the existing custom category fields. For an overview of searching, adding, editing, and deleting items, please see the Control Panel Basics section of this software manual.
Database Fields
Custom Category Fields have the following fields associated with them:
    Name - The name to use for the database field. Field names may contain English letters, numbers, and the underscore (_) character.
    Field Type - The form field type to use for this custom input field.

    Text
    <input type="text">

    Textarea
    <textarea></textarea>

    Checkbox
    <input type="checkbox">

    Select
    <select></select>
    Label - The label to display on web based forms for this field
    Tag Attributes - Extra HTML attributes to use in the HTML tag for this field For example, you may want to specify the size of a text input field in which case you would enter this: size="40" Options - A comma separated list of values to use for the options of a select field
    Validators - The validators to use for this field. Select the validator type (see Custom Field Validators) and the error message that should be displayed when the validation fails. Some validator types require the Extras field to be filled in and that will be noted in the Custom Field Validators documentation.

    Multiple validators can be used for each field. Click the icon to add a validator, or the icon to remove a validator
