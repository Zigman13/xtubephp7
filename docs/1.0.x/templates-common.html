Common Template Variables
Table of Contents
Software Settings
Software settings are always accessed through the $g_config template variable. The following values are available:
    $g_config.site_name - Site Name
    $g_config.meta_description - Meta Description
    $g_config.meta_keywords - Meta Keywords
    $g_config.flag_using_cron - Using cron for stats rollover, conversion queue, etc (1 or 0)
    $g_config.document_root - Document Root
    $g_config.base_url - TubeX URL
    $g_config.base_uri - Virtual path to the TubeX installation
    $g_config.template_uri - Virtual path to the base directory of the selected template set
    $g_config.no_preview - No Thumbnail URL
    $g_config.cookie_domain - Cookie Domain
    $g_config.cookie_path - Cookie Path
    $g_config.flag_mod_rewrite - Use mod_rewrite for search engine friendly URLs (1 or 0)
    $g_config.email_address - E-mail Address
    $g_config.email_name - E-mail Name
    $g_config.date_format - Date Format
    $g_config.time_format - Time Format
    $g_config.dec_point - Decimal Point
    $g_config.thousands_sep - Thousands Separator
    $g_config.avatar_dimensions - Avatar Dimensions
    $g_config.avatar_filesize - Avatar Filesize
    $g_config.avatar_extensions - Avatar Extensions
    $g_config.flag_user_confirm_email - All new user accounts must be confirmed by e-mail
    $g_config.flag_user_strip_tags - Strip HTML tags from submitted account information
    $g_config.video_extensions - File Extensions
    $g_config.video_format - Video Format
    $g_config.video_size - Video Size
    $g_config.video_bitrate - Video Bitrate
    $g_config.audio_bitrate - Audio Bitrate
    $g_config.thumb_size - Thumbnail Size
    $g_config.thumb_quality - Thumbnail Quality
    $g_config.thumb_amount - Thumbnail Amount
    $g_config.flag_allow_uploads - Allow users to upload videos
    $g_config.flag_upload_reject_duplicates - Do not allow the same video file to be uploaded more than once
    $g_config.flag_upload_allow_private - Allow users to make their uploaded videos private
    $g_config.flag_upload_convert - Convert uploaded videos (not recommended for shared servers)
    $g_config.flag_upload_review - Review uploaded videos before displaying on site
    $g_config.upload_extensions - Allowed Extensions
    $g_config.max_upload_size - Maximum Size
    $g_config.max_upload_duration - Maximum Duration
    $g_config.title_min_length - Title Length (minimum)
    $g_config.title_max_length - Title Length (maximum)
    $g_config.description_min_length - Description Length (minimum)
    $g_config.description_max_length - Description Length (maximum)
    $g_config.tags_min - Tags Required (minimum)
    $g_config.tags_max - Tags Required (maximum)
    $g_config.flag_video_strip_tags - Strip HTML tags from submitted video information
    $g_config.comment_max_length - Maximum Length (comments)
    $g_config.comment_throttle_period - Throttle Period
    $g_config.flag_comment_strip_tags - Strip HTML tags from comment text
    $g_config.captcha_min_length - String Length (minimum)
    $g_config.captcha_max_length - String Length (maximum)
    $g_config.flag_captcha_words - Use words file for CAPTCHA strings
    $g_config.flag_captcha_on_signup - Use CAPTCHA on user signup form
    $g_config.flag_captcha_on_upload - Use CAPTCHA on video upload form
    $g_config.flag_captcha_on_comment - Use CAPTCHA on video comment form
    $g_config.cache_main - Main Page (cache)
    $g_config.cache_search - Search Page (cache)
    $g_config.cache_categories - Categories Page (cache)
    $g_config.cache_browse - Browse Videos Page (cache)
    $g_config.cache_video - Video Page (cache)
    $g_config.cache_profile - Profile Page (cache)
    $g_config.cache_comments - Comments Page (cache)
Video
Assuming the video is being accessed through a template variable named $video, the following values are available:
    $video.video_id - The ID number assigned to the video
    $video.username - The username of the account that uploaded the video, if any
    $video.title - The video title
    $video.description - The video description
    $video.tags - The video's tags
    $video.date_added - The date and time the video was added
    $video.date_recorded - The date the video was recorded, if specified
    $video.location_recorded - The location the video was recorded, if specified
    $video.status - The video status (one of Pending, Queued, Active, Disabled)
    $video.duration - The video duration, in seconds
    $video.category_id - The ID number of the category associated with the video
    $video.sponsor_id - The ID number of the sponsor associated with the video
    $video.is_private - Flag indicating if the video is private (1 = private, 0 = not private)
    $video.is_featured - Flag indicating if the video is featured (1 = featured, 0 = not featured)
    $video.date_last_featured - The date and time the video was last featured
    $video.num_thumbnails - The number of thumbnail preview images there are for the video
    $video.display_thumbnail - The ID number of the thumbnail preview image selected for display
    $video.allow_comments - Flag indicating if ratings are allowed (one of No, Yes - Add Immediately, Yes - Require Approval)
    $video.allow_ratings - Flag indicating if ratings are allowed (1 = allowed, 0 = not allowed)
    $video.allow_embedding - Flag indicating if embedding is allowed (1 = allowed, 0 = not allowed)
    $video.is_user_submitted - Flag indicating if the video was submitted by a user (1 = user submitted, 0 = not user submitted)
    $video.conversion_failed - Flag indicating if the video conversion failed (1 = failed, 0 = not failed)
    $video.thumbnail - The URL to the thumbnail preview image for the video
    $video.date_last_view - The date and time the video was last watched
    $video.today_num_views - Number of views today
    $video.today_num_flagged - Number of times flagged today
    $video.today_num_featured - Number of times marked to be featured today
    $video.today_num_comments - Number of comments today
    $video.today_num_favorited - Number of times favorited today
    $video.today_num_ratings - Number of ratings today
    $video.today_sum_of_ratings - Sum of all ratings today
    $video.today_avg_rating - Average of all ratings today
    $video.week_num_views - Number of views this week
    $video.week_num_flagged - Number of times flagged this week
    $video.week_num_featured - Number of times marked to be featured this week
    $video.week_num_comments - Number of comments this week
    $video.week_num_favorited - Number of times favorited this week
    $video.week_num_ratings - Number of ratings this week
    $video.week_sum_of_ratings - Sum of all ratings this week
    $video.week_avg_rating - Average of all ratings this week
    $video.month_num_views - Number of views this month
    $video.month_num_flagged - Number of times flagged this month
    $video.month_num_featured - Number of times marked to be featured this month
    $video.month_num_comments - Number of comments this month
    $video.month_num_favorited - Number of times favorited this month
    $video.month_num_ratings - Number of ratings this month
    $video.month_sum_of_ratings - Sum of all ratings this month
    $video.month_avg_rating - Average of all ratings this month
    $video.total_num_views - Number of views total
    $video.total_num_flagged - Number of times flagged total
    $video.total_num_featured - Number of times marked to be featured total
    $video.total_num_comments - Number of comments total
    $video.total_num_favorited - Number of times favorited total
    $video.total_num_ratings - Number of ratings total
    $video.total_sum_of_ratings - Sum of all ratings total
    $video.total_avg_rating - Average of all ratings total
In addition to the default template variables, the custom database fields that you setup will also be accessible. For example, if you setup a database field named "storage" you will be able to access this value through $video.storage
Clip
Assuming the clip is being accessed through a template variable named $clip, the following values are available:
    $clip.video_id - The ID number of the video this clip is associated with
    $clip.type - A string indicating the clip type, either Embed or URL
    $clip.clip - The clip URL or embed code
    $clip.filesize - The filesize (in bytes) of the clip. Only valid for locally hosted video files.
User
Assuming the user account is being accessed through a template variable named $user, the following values are available:
    $user.username - The account username
    $user.email - The e-mail address associated with the account
    $user.date_created - The date and time the account was created
    $user.status - The user account status (one of Submitted, Pending, Active, or Disabled
    $user.name - The user's name
    $user.date_birth - The user's date of birth
    $user.gender - The user's gender (one of Male or Female)
    $user.relationship - The user's relationship status (one of Single, Taken, or Open)
    $user.about - About the user
    $user.website_url - The user's website URL
    $user.hometown - The user's hometown
    $user.current_city - The user's current city
    $user.postal_code - The user's postal code
    $user.current_country - The user's current country
    $user.occupations - The user's occupations
    $user.companies - Companies the user has worked for
    $user.schools - Schools the user has attended
    $user.hobbies - The user's hobbies
    $user.movies - Movies and TV shows the user likes
    $user.music - Music the user likes
    $user.books - Books the user likes
    $user.uri - The virtual path to the user's avatar, if any
    $user.date_last_login - The date and time the user last logged in
    $user.today_video_views - The number of views this user's videos have received today
    $user.today_profile_views - The number of views this user's profile has received today
    $user.today_videos_watched - The number of videos the user has watched today
    $user.today_bandwidth_used - The bandwidth used by the user today (locally hosted videos only)
    $user.today_videos_uploaded - The number of videos the user has uploaded today
    $user.today_comments_submitted - The number of comments the user has submitted today
    $user.week_video_views - The number of views this user's videos have received this week
    $user.week_profile_views - The number of views this user's profile has received this week
    $user.week_videos_watched - The number of videos the user has watched this week
    $user.week_bandwidth_used - The bandwidth used by the user this week (locally hosted videos only)
    $user.week_videos_uploaded - The number of videos the user has uploaded this week
    $user.week_comments_submitted - The number of comments the user has submitted this week
    $user.month_video_views - The number of views this user's videos have received this month
    $user.month_profile_views - The number of views this user's profile has received this month
    $user.month_videos_watched - The number of videos the user has watched this month
    $user.month_bandwidth_used - The bandwidth used by the user this month (locally hosted videos only)
    $user.month_videos_uploaded - The number of videos the user has uploaded this month
    $user.month_comments_submitted - The number of comments the user has submitted this month
    $user.total_video_views - The number of views this user's videos have received total
    $user.total_profile_views - The number of views this user's profile has received total
    $user.total_videos_watched - The number of videos the user has watched total
    $user.total_bandwidth_used - The bandwidth used by the user total (locally hosted videos only)
    $user.total_videos_uploaded - The number of videos the user has uploaded total
    $user.total_comments_submitted - The number of comments the user has submitted total
In addition to the default template variables, the custom database fields that you setup will also be accessible. For example, if you setup a database field named "storage" you will be able to access this value through $user.storage
Sponsor
Assuming the sponsor is being accessed through a template variable named $sponsor, the following values are available:
    $sponsor.sponsor_id - The ID number associated with the sponsor
    $sponsor.name - The sponsor's name
    $sponsor.url - The sponsor's main site URL
    $sponsor.us2257_url - The sponsor's 2257 URL (useful for adult sites)
    $sponsor.videos - The number of videos associated with the sponsor
In addition to the default template variables, the custom database fields that you setup will also be accessible. For example, if you setup a database field named "storage" you will be able to access this value through $sponsor.storage
Comment
Assuming the comment is being accessed through a template variable named $comment, the following values are available:
    $comment.comment_id - The ID number assigned to the comment
    $comment.video_id - The ID number of the video the comment is associated with
    $comment.username - The username of the account that submitted the comment
    $comment.date_commented - The date and time the comment was submitted
    $comment.status - The comment status (will only be Active)
    $comment.comment - The comment itself
Category
Assuming the category is being accessed through a template variable named $category, the following values are available:
    $category.category_id - The ID number assigned to the category
    $category.name - The name of the category
    $category.url_name - The URL name of the category
    $category.description - The description assigned to the category
    $category.meta_description - The meta description assigned to the category
    $category.meta_keywords - The meta keywords assigned to the category
    $category.auto_category_term - The auto category term value assigned to the category
    $category.num_videos - The number of active videos in the category
    $category.uri - If the category has an associated image, the virtual path to that image for use in an <img> tag
In addition to the default template variables, the custom database fields that you setup will also be accessible. For example, if you setup a database field named "storage" you will be able to access this value through $category.storage
Rating
Assuming the rating is being accessed through a template variable named $rating, the following values are available:
    $rating.username - The username of the account that submitted the rating
    $rating.video_id - The ID number of the video associated with the rating
    $rating.rating - The rating the user gave
    $rating.date_rated - The date and time the rating was submitted
Reason
Assuming the reason is being accessed through a template variable named $reason, the following values are available:
    $reason.reason_id - The ID number assigned to the reason
    $reason.type - The type of reason (either Flag or Feature)
    $reason.short_name - The short name assigned to the reason
    $reason.description - The description assigned to the reason
Search Term
Assuming the search term is being accessed through a template variable named $term, the following values are available:
    $term.term_id - The ID number assigned to the search term
    $term.term - The search term itself
    $term.frequency - The number of times the search term has been used
Tag
Assuming the tag is being accessed through a template variable named $tag, the following values are available:
    $tag.tag - The tag itself
    $tag.frequency - The number of times the tag is used in currently active videos
