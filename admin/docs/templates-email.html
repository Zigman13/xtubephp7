Template Editing Interface
Editing the e-mail templates is done through the Templates > E-mail Templates function in the control panel. That interface will look something like this:

    Available Templates - This section will list all of the available site templates that you can edit. When you first access the page, only this section will be shown. Click on a template name to load that template for editing.
    Message Subject - Enter the subject you would like to use for the e-mail message
    Formatting Controls - Use the formatting controls to adjust how text will appear in the message body. There are controls to center, bold, adjust font, etc.
    Message Body - Enter the body of the e-mail message
    Toolbar Functions - Use the toolbar functions to peform the following actions:

    Click this icon to bring up the Search and Replace dialog where you can perform a search and replace on many templates at once.
    Click this icon to save the currently loaded template
    Click this icon to reload the template code for the currently loaded template. This is useful when you do a search and replace which affects the current template you are editing.
    Click this icon to access this documentation file
Search and Replace
TubeX allows you to make quick changes to your templates in bulk with the Search and Replace function. When you access the Search and Replace function a dialog will appear that will allow you to select the templates that you want to perform this action on, the code to search for in the template, and the replacement code to use. Multiple templates can be selected by holding down the shift or ctrl keys on your keyboard as you click on templates in the selection list. Once you have selected the templates and entered the search and replacement code, press the Apply Changes button to start the task. Once completed a notification message will be displayed letting you know how many replacements were made.
Greeting and Signature Templates
There are two special e-mail templates that you can edit which only allow configuring of the Message Body and do not have a Subject. Those are:
    email-global-greeting.tpl - Use this template to control the text that is appended to the beginning of every e-mail message sent by the software. This is a good place to put a simple greeting.
    email-global-signature.tpl - Use this template to control the text that is appended to the end of every e-mail message sent by the software. This is a good place to put your signature, a privacy notice, a link to your site, etc.
Templates & Their Variables
All of the e-mail templates will have the following two variables set:
    $g_config - The software settings (see Common Template Variables)
    $g_user - The user account settings (see Common Template Variables)
Below you will find a list of all the e-mail templates shipped with the software, a description of when the template is used, and details on the template variables available (if any) on that template.

email-user-added.tpl
    This e-mail message will be sent to a user after they add their account, but only if you are not requiring e-mail confirmation or reviewing new accounts.
email-user-approved.tpl
    This e-mail message will be sent to a user when their account is approved by an administrator (when the Review new user accounts software setting is enabled).
email-user-confirm.tpl
    This e-mail message will be sent to a user after they signup, but only if the All new user accounts must be confirmed by e-mail software setting is enabled.
email-user-pending.tpl
    This e-mail message will be sent to a user after they signup, but only if the Review new user accounts software setting is enabled.
email-user-rejected.tpl
    This e-mail message will be sent to a user when their account is rejected by an administrator (when the Review new user accounts software setting is enabled). The $g_user variable will contain one additional value if you selected a rejection reason:

    $g_user.reject_reason - The Description value of the rejection reason
email-user-reset-confirm.tpl
    This e-mail message will be sent to a user when they use the lost password form. The message will contain a link that the user must click on to confirm that they requested their password be reset.

    $g_code - The confirmation code that will need to be contained in the link the user clicks
email-user-reset.tpl
    This e-mail message will be sent to a user when they have confirmed their lost password request by clicking on the confirmation link supplied in the e-mail message they received.

    $g_password - The new password that has been generated for this user account
