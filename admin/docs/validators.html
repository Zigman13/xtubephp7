Available Validators
The following validators are available for custom database fields:
    NONE - No validation will be performed (no Extras)
    IS_EMPTY - Validate that the field is empty (no Extras)
    NOT_EMPTY - Validate that the field is not empty (no Extras)
    GREATER - Validate that the field is greater than the value you enter for Extras
    GREATER_EQ - Validate that the field is greater than or equal to the value you enter for Extras
    LESS - Validate that the field is less than the value you enter for Extras
    LESS_EQ - Validate that the field is less than or equal to the value you enter for Extras
    VALID_EMAIL - Validate that the field contains a valid e-mail address (no Extras)
    VALID_HTTP_URL - Validate that the field contains a valid HTTP URL (no Extras)
    LENGTH_LESS - Validate that the string length of the field is less than the value you enter for Extras
    LENGTH_LESS_EQ - Validate that the string length of the field is less than or equal to the value you enter for Extras
    LENGTH_GREATER - Validate that the string length of the field is greater than the value you enter for Extras
    LENGTH_GREATER_EQ - Validate that the string length of the field is greater than or equal to the value you enter for Extras
    IS_ZERO - Validate that the field is exactly zero (no Extras)
    IS_ALPHANUM - Validate that the field contains only alpha-numeric characters (no Extras)
    IS_NUMERIC - Validate that the field contains only numeric characters (no Extras)
    IS_TRUE - Validate that the field is exactly PHP's true value (no Extras)
    IS_FALSE - Validate that the field is exactly PHP's false value (no Extras)
    NOT_TRUE - Validate that the field is anything but PHP's true value (no Extras)
    NOT_FALSE - Validate that the field is anything but PHP's false value (no Extras)
    REGEX_MATCH - Validate that the field matches the regular expression you enter for Extras
    REGEX_NO_MATCH - Validate that the field does not match the regular expression you enter for Extras
    IS_BETWEEN - Validate that the field is between the two values (comma seprated) that you enter for Extras
    VALID_DATETIME - Validate that the field is a valid datetime value (YYYY-MM-DD HH:MM:SS) (no Extras)
    VALID_DATE - Validate that the field is a valid date value (YYYY-MM-DD) (no Extras)
    VALID_TIME - Validate that the field is a valid time value (HH:MM:SS) (no Extras)
    CONTAINS - Validate that the field contains the string you enter for Extras
    NOT_CONTAINS - Validate that the field does not contain the string you enter for Extras
    LENGTH_BETWEEN - Validate that the string length of the field is between the two values (comma seprated) that you enter for Extras
    NOT_ZERO - Validate that the field is not exactly zero (no Extras)
    EQUALS - Validate that the field equals the value you enter for Extras
    NOT_EQUALS - Validate that the field does not equal the value you enter for Extras
